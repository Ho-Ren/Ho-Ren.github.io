User Interviews

User interviews are always important to get an unbiased opinion to questions the interviewer may have. The questions I asked were mainly suggestions on improvements of the overall website and their initial thoughts on the aesthetics.

Competitor Analysis

Competitor analysis is used to identify competitors and see what makes their product strong and what potential weaknesses are present. Determining strengths and weaknesses on other portfolio sites can be difficult because everybody has different tastes but there are websites that are just universally good. I reviewed fellow UCI student Chris Wong's personal website as it was also built from scratch which shows that Chris is does not need to rely on existing frameworks to make his website look amazing.

Feature Value Matrix

A feature value matrix is a chart that determines the value of a particular feature for in terms of business, technical, and user value. The features I proposed were several different ways to contact me in the form of email, LinkedIn, and even GitHub. Giving other people options is always a good thing to have and giving recruiters an option to see my LinkedIn and GitHub profiles allows them to find more information about me in a professional setting.

Feature Prioritization

Feature prioritization is the phase in which I prioritized the many different features I had in mind from the feature value matrix. Contact information has the highest priority simply because the purpose of this website is to not only show what kind of person I am and the work I have done, but also to have the user contact me for potential career opportunities. The contact information was placed at the bottom of the page after all the content.

User Testing:

User Stories

A user story is a great method to learn what a potential user's expectations are and how they might use a certain product. My user stories are about recruiters, colleagues, and hiring managers because they are the main audience of my website.

Cognitive Walkthrough

A cognitive walkthrough is a process in which a user go through a series of tasks to demonstrate usability of a product. For my website, I conducted three cognitive walkthroughs with fellow classmates and colleagues. Many of their answers to my questions were expected.

User Test

Questions Test

Click Test

A user test is used to see how different people use a certain product. I used a click test to see if people could easily figure out how to get my contact information. I also used a question test to see if users had similar impressions on the website. It was quite challenging to get the question test results to be consistent because of the limited amount of people testing as well as the overall openness of the questions.

Summary of Findings

Conducting both user testing and user research, I was able to learn more about the process behind creating a project from scratch. I learned to focus on design and structure first before actually implementing anything so I can get good feedback on aesthetics and usability. The user interviews and cognitive walkthroughs were both useful as the users gave great suggestions on what to add on my website.
